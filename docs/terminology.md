# Terminology

This document defines the standard terminology used throughout the nix-config repository.

## Core Concepts

### System

**Definition**: The operating system name (darwin, nixos)

**Usage**:

- Directory structure: `system/darwin/`, `system/nixos/`, `system/shared/`
- Function parameters: `system = "darwin"` or `system = "nixos"`
- Variable names: `actualSystem`, `systemName`

**Examples**:

```nix
# Correct
system = "darwin";
system = "nixos";

# Incorrect
platform = "darwin";  # Wrong - this would be the Nix platform string
```

### Architecture

**Definition**: The CPU architecture (aarch64, x86_64)

**Usage**:

- Host configuration field: `architecture = "aarch64";`
- Declared in each host's `default.nix`
- Platform-agnostic (same value on darwin and nixos)

**Available values**:

- `"aarch64"` - ARM 64-bit (Apple Silicon, modern ARM servers)
- `"x86_64"` - Intel/AMD 64-bit (most desktops, laptops, servers)

**Examples**:

```nix
# In host configuration
{
  name = "home-macmini-m4";
  architecture = "aarch64";  # Apple Silicon M4
  ...
}
```

### Platform

**Definition**: The Nix platform string combining architecture + system

**Format**: `{architecture}-{system_suffix}`

- Darwin: `{architecture}-darwin` (e.g., "aarch64-darwin")
- Linux/NixOS: `{architecture}-linux` (e.g., "x86_64-linux")

**Usage**:

- Nix system attribute: `nixpkgs.lib.nixosSystem { system = platform; }`
- Variable names: `platform`, `nixPlatform`
- Generated by: `configLoader.getNixPlatform { host; system; }`

**Examples**:

```nix
# Correct
platform = "aarch64-darwin";
platform = "x86_64-linux";

# Incorrect
system = "aarch64-darwin";  # Wrong - system is just "darwin" or "nixos"
```

## Function Naming Conventions

### config-loader.nix

```nix
# Get the system name (darwin/nixos) for a host
getPlatformForHost = { hostName }: "darwin" | "nixos"

# Get the Nix platform string for a host
getNixPlatform = { host, system ? null }: "aarch64-darwin" | "x86_64-linux" | ...

# Load host configuration
loadHost = { host, system ? null }: { hostData }
```

### Platform Libraries

```nix
# darwin.nix / nixos.nix
mkDarwinConfig = { user, host }: let
  # Get Nix platform string (architecture + system)
  platform = configLoader.getNixPlatform { inherit host; system = "darwin"; };
  
  # Use platform for nixpkgs
  inputs.nix-darwin.lib.darwinSystem {
    system = platform;  # e.g., "aarch64-darwin"
    ...
  };
```

## Migration Notes

**Old terminology** (inconsistent):

- ❌ `platform` used for system name ("darwin", "nixos")
- ❌ `system` used for platform string ("aarch64-darwin")

**New terminology** (consistent):

- ✅ `system` = OS name ("darwin", "nixos")
- ✅ `architecture` = CPU arch ("aarch64", "x86_64")
- ✅ `platform` = Nix platform string ("aarch64-darwin", "x86_64-linux")
