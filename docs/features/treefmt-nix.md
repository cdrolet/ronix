# treefmt-nix Integration

Multi-language code formatting for the nix-config repository.

**Added**: 2025-12-16
**Status**: Active

## Overview

treefmt-nix provides unified formatting across all file types in the repository using a single command. It wraps [treefmt](https://github.com/numtide/treefmt) with Nix-native configuration.

## Supported Formatters

| Language | Formatter | File Patterns |
|----------|-----------|---------------|
| Nix | [alejandra](https://github.com/kamadorueda/alejandra) | `*.nix` |
| Markdown | [mdformat](https://github.com/executablebooks/mdformat) | `*.md` |
| Shell | [shfmt](https://github.com/mvdan/sh) | `*.sh` |
| JSON | [prettier](https://prettier.io/) | `*.json` |
| YAML | [prettier](https://prettier.io/) | `*.yaml`, `*.yml` |
| TOML | [taplo](https://github.com/tamasfe/taplo) | `*.toml` |

## Usage

### Format All Files

```bash
# Via justfile (recommended)
just fmt

# Via nix directly
nix fmt
```

### Check Formatting (CI)

```bash
# Fails if any files need formatting
just fmt-check

# Or directly
nix fmt -- --fail-on-change
```

### Format Specific Files

```bash
# treefmt can target specific paths
nix fmt -- path/to/file.nix
```

## Configuration

Configuration is in `treefmt.nix` at the repository root:

```nix
{ ... }: {
  projectRootFile = "flake.nix";

  # Nix formatter
  programs.alejandra = {
    enable = true;
    excludes = [
      "specs/*"  # Design documents with pseudo-code
    ];
  };

  # Markdown formatter
  programs.mdformat.enable = true;

  # Shell script formatter
  programs.shfmt = {
    enable = true;
    indent_size = 2;
  };

  # JSON/YAML formatter
  programs.prettier = {
    enable = true;
    includes = [ "*.json" "*.yaml" "*.yml" ];
    excludes = [ "flake.lock" ];
  };

  # TOML formatter
  programs.taplo.enable = true;
}
```

## Exclusions

### Nix Files

The `specs/` directory is excluded from Nix formatting because it contains design documents with example/pseudo-code that may not be valid Nix syntax.

### JSON Files

`flake.lock` is excluded from prettier formatting as it's auto-generated by Nix.

## Adding New Formatters

treefmt-nix supports 120+ formatters. To add one:

1. Edit `treefmt.nix`
1. Enable the program:
   ```nix
   programs.black.enable = true;  # Python formatter
   ```
1. Run `nix fmt` to verify

Available formatters: [treefmt-nix options](https://github.com/numtide/treefmt-nix#supported-programs)

## Architecture

```
flake.nix
├── inputs.treefmt-nix          # Flake input
├── treefmtEval                  # Evaluates treefmt.nix per system
└── formatter.${system}          # Exposes wrapper for nix fmt

treefmt.nix                      # Formatter configuration
```

The formatter is generated for all supported systems:

- `aarch64-darwin` (Apple Silicon)
- `x86_64-darwin` (Intel Mac)
- `x86_64-linux`
- `aarch64-linux`

## Integration with Editor

### Helix

The existing helix configuration already uses alejandra for Nix files:

```nix
# system/shared/app/editor/helix.nix
languages.language = [{
  name = "nix";
  formatter.command = "${pkgs.alejandra}/bin/alejandra";
}];
```

This remains unchanged - treefmt-nix handles repository-wide formatting while editors handle on-save formatting.

### VS Code

Add to settings.json:

```json
{
  "editor.formatOnSave": true,
  "[nix]": {
    "editor.defaultFormatter": "kamadorueda.alejandra"
  }
}
```

## CI Integration

For GitHub Actions or similar:

```yaml
- name: Check formatting
  run: nix fmt -- --fail-on-change
```

Or using justfile:

```yaml
- name: Check formatting
  run: just fmt-check
```

## Troubleshooting

### "path does not exist" error

Ensure new files are tracked by git:

```bash
git add treefmt.nix
nix fmt
```

### Formatter fails on specific file

Check if the file has syntax errors. For Nix files:

```bash
nix-instantiate --parse path/to/file.nix
```

### Exclude a file from formatting

Add to the appropriate formatter's `excludes` in `treefmt.nix`:

```nix
programs.alejandra.excludes = [
  "path/to/excluded/file.nix"
];
```
